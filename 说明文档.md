Oasis 项目说明文档（深入浅出版）

更新时间：2026-01-05

一、为什么需要 Oasis
- 真实平台不可控：在 Twitter/Reddit 等真实平台上做实验，受限于接口、政策与不可重复的外部环境，难以做课堂教学与系统性研究。Oasis 提供一个“可控、可复现”的社交环境。
- 研究与教学一体化：信息传播、推荐算法、群体行为、图分析、LLM 工具调用等主题往往跨越多个学科。Oasis 把这些模块拼在一起，让课程/论文能在同一个平台上跑通。
- 可审计与复现：所有行动都会写入数据库与轨迹表（trace）。你可以重放、检查、对比不同策略，保证实验可复现与结果可解释。
- 低门槛起步：使用 Python 异步和 SQLite 起步即可，Neo4j、TWHIN-BERT 等都是“可选增强件”，适合先跑通后逐步升级。

二、为什么这样设计
- 三层结构（Agent / Platform / Channel）：  
  - 想要：让“代理的意图”与“平台的执行”解耦，避免耦在一起难以替换。  
  - 做法：Agent 只负责决定做什么；Platform 负责把动作落地到数据库与规则；Channel 只传递消息。这样替换推荐算法或添加新行动不会影响其他层。
- 异步模型（asyncio）：  
  - 想要：同一仿真步中让许多代理并发行动，且轻量可控。  
  - 做法：用协程和信号量控制并发，避免进程/线程的重资源开销与复杂同步。
- SQLite + 可选 Neo4j：  
  - 想要：本地即用、零运维；需要图查询或大规模关系时再升级。  
  - 做法：默认 SQLite 管核心数据；需要图分析/可视化时启用 Neo4j 或 igraph，不强制绑定重型后端。
- 多种推荐策略：  
  - 想要：从随机/热门到个性化语义推荐，覆盖课堂与研究常见场景。  
  - 做法：内置随机、Reddit 热度、Twitter 风格语义个性化、TWHIN-BERT 等，方便横向对比与扩展。
- 统一时间语义（Clock）：  
  - 想要：不同平台规则依赖时间（热度衰减、趋势窗口）。  
  - 做法：Twitter 用步进时间、Reddit 用真实时间映射，保证规则可解释且可调。

三、你可以用它做什么
- 教学演示：几分钟跑通一个“社交平台”小世界，观察推荐、点赞、转发如何影响传播。
- 实验研究：对比不同推荐策略的影响；分析关注网络结构与群体极化；跑 LLM 工具调用的可审计对话。
- 快速原型：把你自己的行动/规则嵌入平台，用真实脚本批量生成用户与数据，验证想法。

四、核心模块（白话版）
1. 环境模块（Environment）
- 负责“每一步仿真怎么走”：reset/step/close 三件事。step 会先更新推荐，再让所有代理一起行动，最后把结果写回平台和数据库。
- 入口：OasisEnv 类 [env.py](file:///d:/GitProj/oasis-main/oasis/environment/env.py)  
  - reset：启动平台后台循环、注册/生成代理  
  - step：刷新推荐 → 并发执行代理行动（可手动或由 LLM 决策）  
  - close：发退出信号、收尾数据库  
- 动作类型定义：[env_action.py](file:///d:/GitProj/oasis-main/oasis/environment/env_action.py)  
  - ManualAction：提前写好“做什么、带什么参数”的行动  
  - LLMAction：让大模型先读环境文本，再决定调用哪些工具函数
- 文本环境生成：[agent_environment.py](file:///d:/GitProj/oasis-main/oasis/social_agent/agent_environment.py)  
  - 把“推荐帖、关注关系、群聊消息”等整理成易读的提示，供 LLM 决策。

2. 代理模块（Social Agent）
- 负责“每个用户像什么样、会做什么事”。用工具函数表示可做的动作（发帖、点赞、关注、群聊、购物等）。
- 主要文件：  
  - 代理类：[agent.py](file:///d:/GitProj/oasis-main/oasis/social_agent/agent.py)  
    - perform_action_by_llm：读环境 → 让模型决定 → 执行工具调用  
    - perform_action_by_data：直接按函数名执行，适合脚本化实验  
    - perform_interview：做访谈型测试、记录想法  
    - perform_agent_graph_action：把关注/取关同步到关系图  
  - 行动集合：[agent_action.py](file:///d:/GitProj/oasis-main/oasis/social_agent/agent_action.py)  
    - 用 FunctionTool 暴露异步行动，所有行动通过 Channel 请求 Platform 并等待结果  
  - 关系图：[agent_graph.py](file:///d:/GitProj/oasis-main/oasis/social_agent/agent_graph.py)  
    - 可选 igraph/Neo4j，支持添加/删除节点边、统计、可视化  
  - 批量生成代理：[agents_generator.py](file:///d:/GitProj/oasis-main/oasis/social_agent/agents_generator.py)  
    - 从 CSV/JSON 批量创建、建关注、导入历史帖子；支持大规模与可控代理

3. 平台模块（Social Platform）
- 负责“把动作真正落地到数据库与规则里”，并维护推荐与时间。
- 主要文件：  
  - 消息通道：[channel.py](file:///d:/GitProj/oasis-main/oasis/social_platform/channel.py)  
    - 生产者-消费者队列；代理写请求，平台读并返回结果  
  - 平台核心：[platform.py](file:///d:/GitProj/oasis-main/oasis/social_platform/platform.py)  
    - 运行循环：持续消费行动请求、路由到具体异步函数  
    - 数据库：启动时创建/连接 SQLite  
    - 行动实现：发帖、转发、引用、点赞/取消、点踩/取消、搜索、趋势、关注/取关、静音/取消、评论、举报、商品注册与购买、群聊等  
    - 推荐更新：随机/Reddit 热度/Twitter 语义/TWHIN-BERT  
    - 时间管理：Clock 控制步进或真实时间映射  
  - 数据库与表结构：[database.py](file:///d:/GitProj/oasis-main/oasis/social_platform/database.py)、schema/*.sql  
    - 自动创建常用表；提供读写与推荐缓存抓取/回填  
  - 平台工具：[platform_utils.py](file:///d:/GitProj/oasis-main/oasis/social_platform/platform_utils.py)  
    - 通用 SQL、行为轨迹记录、评论聚合、帖子类型解析、自评限制  
  - 推荐系统：[recsys.py](file:///d:/GitProj/oasis-main/oasis/social_platform/recsys.py)  
    - 从随机到个性化到 TWHIN-BERT 的多种实现  
  - 类型定义：[typing.py](file:///d:/GitProj/oasis-main/oasis/social_platform/typing.py)  
    - 行动枚举（ActionType）、推荐系统枚举（RecsysType）等

五、如何快速上手（保姆级）
- 准备环境：Python 版本满足项目要求；可选安装 Neo4j（做图查询/可视化时用）。  
- 拉起平台：在 [oasis/environment/env.py](file:///d:/GitProj/oasis-main/oasis/environment/env.py) 选择默认平台类型（Twitter/Reddit）或传入自定义 Platform。  
- 生成代理与数据：用 [agents_generator.py](file:///d:/GitProj/oasis-main/oasis/social_agent/agents_generator.py) 或 generator 目录脚本批量创建用户、关注关系、历史帖子。  
- 运行仿真：调用 OasisEnv 的 reset → 多次 step → close；观察数据库变化与日志输出。  
- 分析结果：使用 [visualization](file:///d:/GitProj/oasis-main/visualization) 的脚本生成图表或统计指标。

六、常见疑问（FAQ）
- 为什么不直接在真实平台做实验？  
  - 真实平台不可控且不可复现，Oasis 提供可控环境与完整数据留痕，适合教学与研究。
- 为什么选 SQLite 而不是分布式数据库？  
  - 教学与中小规模实验以“易用、零运维”为先。需要扩展时可以迁移到更强后端。
- LLM 工具调用如何保证可审计？  
  - 所有工具调用写入 trace 表；参数、时间、执行结果都有记录，支持复盘与比较。
- 推荐系统可以自定义吗？  
  - 可以。在 [recsys.py](file:///d:/GitProj/oasis-main/oasis/social_platform/recsys.py) 增加策略并在 Platform 选择即可。
- Neo4j 是否必须？  
  - 不是。默认用 igraph 完成统计与绘图；需要图数据库查询/可视化时再启用 Neo4j。

七、设计取舍与扩展建议
- 保持“意图与执行”分离：新增行动优先在 Agent 侧以工具函数暴露，再在 Platform 落地实现。
- 先简单再复杂：先用随机/Reddit 热度把流程跑通，再替换为个性化或 TWHIN-BERT。
- 按需开启重型组件：只有当分析需求明确时再接入 Neo4j、嵌入模型、外部服务。

八、术语小结
- Agent：具备画像与工具函数的社交用户实体，可能由大模型驱动。  
- Platform：动作实现与数据存储、推荐与规则的统一入口。  
- Channel：代理与平台的异步消息通道。  
- Trace：行为轨迹表，记录行动及参数与时间。  
- Recsys：推荐算法模块，生成推荐缓冲供展示与决策。  
- Graph：关系网络，关注/取关等行为映射为有向边。

三、核心模块详解

1. 环境模块（Environment）
- 作用：负责仿真调度（reset/step/close）、并发控制（LLM 并发信号量）、与平台交互（推荐表刷新、执行行动），并为代理提供“观察到的环境文本”。
- 入口与职责：
  - OasisEnv 类：[env.py](file:///d:/GitProj/oasis-main/oasis/environment/env.py)  
    - reset：启动平台后台协程、注册或生成代理；  
    - step：在每个仿真步，先刷新推荐系统，再并发执行各代理的行动（支持手动与 LLM 行动）；  
    - close：发送退出信号，收尾数据库连接。  
    - 支持默认平台类型（Twitter/Reddit）或自定义 Platform 实例，初始化时自动配置推荐系统参数与 Channel。
  - 环境动作类型定义：[env_action.py](file:///d:/GitProj/oasis-main/oasis/environment/env_action.py)  
    - ManualAction：手动预定义行动（带参数）；  
    - LLMAction：由大模型推理产生的行动占位类型（配合工具函数实现）。
  - 文本环境生成：[agent_environment.py](file:///d:/GitProj/oasis-main/oasis/social_agent/agent_environment.py)  
    - 将“推荐帖子、关注/粉丝、群聊消息”等转为可读提示，供 LLM 决策。

2. 代理模块（Social Agent）
- 作用：定义智能体形象与能力，管理其工具函数（行动）与记忆，并与 Channel/Platform 协作执行社会化行为。
- 主要文件：
  - 代理类：[agent.py](file:///d:/GitProj/oasis-main/oasis/social_agent/agent.py)  
    - 继承 ChatAgent（来自 CAMEL 框架），构造时注入用户画像（UserInfo）、工具函数列表（如点赞、关注、发帖、群聊等），支持最大迭代数与访谈记录；  
    - perform_action_by_llm：观察环境、提交模型调用、解析工具调用并执行；  
    - perform_action_by_data：按函数名直接调用工具，实现“脚本化行为”；  
    - perform_interview：用于访谈测试与记录；  
    - perform_agent_graph_action：将关注/取关行为映射到关系图。
  - 行动集合：[agent_action.py](file:///d:/GitProj/oasis-main/oasis/social_agent/agent_action.py)  
    - 使用 FunctionTool 将异步行动封装成可供 LLM 函数调用的接口，涵盖发帖、点赞/点踩、评论、搜索、趋势、关注/取关、静音/取消静音、群聊、举报、购买商品等。  
    - 各行动通过 Channel 队列向 Platform 发送请求，并等待返回。
  - 关系图与 Neo4j 支持：[agent_graph.py](file:///d:/GitProj/oasis-main/oasis/social_agent/agent_graph.py)  
    - 后端可选 igraph 或 Neo4j；  
    - 提供添加/删除节点与边、查询统计与可视化（igraph 绘图）。  
  - 批量生成代理：[agents_generator.py](file:///d:/GitProj/oasis-main/oasis/social_agent/agents_generator.py)  
    - 从 CSV/JSON 批量创建代理（含注册、建立关注关系、导入历史帖子）；  
    - 支持百万规模优化、可控代理、Reddit 风格代理生成等。

3. 平台模块（Social Platform）
- 作用：负责全链路的“平台级”逻辑，包括：数据库读写、推荐系统更新、行动执行、群聊与举报管理、时间推进等。
- 主要文件：
  - 消息通道：[channel.py](file:///d:/GitProj/oasis-main/oasis/social_platform/channel.py)  
    - 基于 asyncio 的生产者-消费者队列；代理写入行动请求，平台读取并返回结果。
  - 平台核心：[platform.py](file:///d:/GitProj/oasis-main/oasis/social_platform/platform.py)  
    - running：循环消费行动请求，路由到具体异步函数；  
    - 数据库：启动时创建/连接 SQLite；  
    - 行动实现：发帖、转发、引用、点赞/取消点赞、点踩/取消点踩、搜索、趋势、关注/取关、静音/取消静音、评论相关、举报、商品注册与购买、群聊（创建/加入/退出/发送/监听）等；  
    - 推荐系统更新：根据 RecsysType 选择策略（随机/Reddit 热度/Twitter 语义/TWHIN-BERT）；  
    - 时间管理：Clock 控制步进或真实时间映射。
  - 数据库与表结构：[database.py](file:///d:/GitProj/oasis-main/oasis/social_platform/database.py)、schema/*.sql  
    - 自动创建表：user、post、follow、mute、like、dislike、report、trace、rec、comment、comment_like、comment_dislike、product、group、group_member、group_message；  
    - 提供表读取、推荐缓存矩阵抓取与回填。
  - 平台工具：[platform_utils.py](file:///d:/GitProj/oasis-main/oasis/social_platform/platform_utils.py)  
    - 封装通用 SQL 操作、行为轨迹记录、评论聚合、帖子类型解析（普通/转发/引用）、自评限制（不允许点赞/点踩自己的帖/评）等。
  - 推荐系统：[recsys.py](file:///d:/GitProj/oasis-main/oasis/social_platform/recsys.py)  
    - rec_sys_random：随机推荐；  
    - rec_sys_reddit：按照 Reddit 热度分数挑选热门帖；  
    - rec_sys_personalized / rec_sys_personalized_with_trace：基于用户简介与帖子语义嵌入计算相似度，结合交互轨迹优化；

四、设计动机与功能实现串讲
- 分层解耦：Agent 只负责“决策与意图”，Platform 负责“行动落地与数据记录”，Channel 负责“消息传递”。这种三层结构便于替换与扩展（如换推荐算法、换消息通道、接入不同平台规则）。
- 工具函数驱动行动：通过 FunctionTool 将行动暴露为“函数调用”，LLM 可以在一次对话中规划并触发多个行动，形成“可审计”的轨迹（trace 表）。
- 推荐系统与行为闭环：每个仿真步先更新推荐缓存，再让代理观察环境并行动，行动进一步写回数据库（如点赞数、分享数），影响下一步的推荐与决策，实现闭环。
- 关系网络与图分析：AgentGraph 使用 igraph 或 Neo4j，关注/取关映射为有向边，便于研究传播路径、社群结构与极化现象。
- 时间语义与规则：Clock 统一仿真时间，支持 Reddit/Twitter 两种时间观，趋势榜、报告阈值等规则依赖时间推进。

五、运行流程总览（教学视角）
- 初始化：选择平台类型（Twitter/Reddit）并创建 Platform；初始化 Channel 与 SQLite 数据库；设置推荐系统参数。
- 生成代理：读取 CSV/JSON，批量注册用户、建立关注关系、导入历史帖子；或创建少量“可控代理”用于课堂演示。
- 启动仿真：环境 reset 后，Platform 后台任务开始循环消费代理消息；每步 step 先刷新推荐，再让各代理执行行动（手动/LLM）。
- 收尾与分析：close 发送退出指令，数据库落盘；通过 visualization 脚本分析结果或导出图形。

八、术语小结
- Agent：具备画像与工具函数的社交用户实体，可能由大模型驱动。

- Channel：代理与平台的异步消息通道。
- Trace：行为轨迹表，记录用户在何时执行了何种行动及其参数。
- Recsys：推荐算法模块，生成 rec 推荐缓冲，供 refresh 展示。
- Graph：关系网络，关注/取关等行为映射为有向边。
